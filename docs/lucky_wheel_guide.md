# 幸运转盘活动功能说明

## 功能概述

幸运转盘活动是一个完整的抽奖系统，支持多个转盘配置，每个转盘包含固定8个奖项和动态规则配置。

## 系统架构

### 数据库表结构

1. **幸运转盘主配置表** (`slot_lucky_wheel_config`)
   - 活动标题
   - Banner图
   - 打码倍数
   - 活动状态

2. **转盘表** (`slot_lucky_wheel_turntable`)
   - 转盘名称
   - 解锁条件（充值金额）
   - 赠送次数
   - 奖项配置（JSON格式）
   - 规则配置（JSON格式）
   - 转盘状态

3. **转盘记录表** (`slot_lucky_wheel_logs`)
   - 用户ID
   - 转盘ID
   - 中奖奖项标题
   - 中奖金额
   - 发放状态

## 功能特性

### 1. 主配置管理
- **活动标题**：设置活动名称
- **Banner图**：上传活动宣传图片
- **打码倍数**：设置提现打码倍数（0.1-10倍）
- **活动状态**：启用/禁用整个活动

### 2. 转盘配置
每个转盘包含以下配置：

#### 基础配置
- **转盘名称**：转盘显示名称
- **解锁条件**：用户需要充值多少金额才能解锁该转盘
- **赠送次数**：默认赠送的转盘次数
- **用户最大次数**：限制每个用户使用该转盘的最大次数（0表示无限制）
- **转盘状态**：启用/禁用该转盘

#### 奖项配置（动态数量）
- **奖项标题**：奖项显示名称
- **金额**：奖项金额（0表示谢谢参与）
- **中奖概率**：0-1之间的小数（概率总和不再限制为1，系统会自动计算总概率并按比例抽奖）
- **排序**：奖项在转盘上的位置（1-N，N为奖项总数）

#### 规则配置（动态添加）
- **规则类型**：
  - 充值达到：用户充值达到指定金额
- **条件值**：触发条件的具体数值
- **赠送次数**：满足条件后赠送的转盘次数
- **规则状态**：启用/禁用该规则

## 使用流程

### 1. 初始化数据库
```sql
-- 执行 database/lucky_wheel_tables.sql 创建表结构
```

### 2. 配置活动
1. 进入后台管理 → 活动管理 → 幸运转盘 → 配置
2. 设置活动标题、Banner图、打码倍数
3. 保存配置

### 3. 配置转盘
1. 进入转盘1/2/3配置页面
2. 设置转盘基础信息（名称、解锁条件、赠送次数）
3. 配置8个奖项（标题、金额、概率、排序）
4. 添加转盘规则（可选）
5. 保存配置

## 技术实现

### 后端架构
- **框架**：ThinkPHP 8.1
- **数据库**：MySQL
- **模型层**：
  - `LuckyWheelConfig`：主配置模型
  - `LuckyWheelTurntable`：转盘模型
  - `LuckyWheelLogs`：记录模型
- **控制器层**：
  - `LuckyWheel`：主配置控制器
  - `LuckyWheelTurntable`：转盘管理控制器

### 前端架构
- **框架**：Vue 3 + TypeScript
- **UI库**：Element Plus
- **路由**：Vue Router
- **状态管理**：Pinia

### 数据存储
- 奖项和规则使用JSON格式存储在转盘表中
- 支持复杂的配置数据结构
- 便于扩展和维护

## API接口

### 主配置接口
- `GET /admin/activity.lucky_wheel/edit` - 获取主配置
- `POST /admin/activity.lucky_wheel/edit` - 保存主配置

### 转盘管理接口
- `GET /admin/activity.lucky_wheel_turntable/edit` - 获取转盘配置
- `POST /admin/activity.lucky_wheel_turntable/edit` - 保存转盘基础配置
- `POST /admin/activity.lucky_wheel_turntable/updatePrizes` - 更新奖项配置
- `POST /admin/activity.lucky_wheel_turntable/updateRules` - 更新规则配置
- `GET /admin/activity.lucky_wheel_turntable/getUserUsage` - 获取用户转盘使用情况
- `GET /admin/activity.lucky_wheel_turntable/getStatistics` - 获取转盘统计信息

## 注意事项

1. **奖项数量**：奖项数量不再固定为8个，可以根据需要添加或删除
2. **奖项概率**：奖项概率不再限制总和为1，系统会自动计算总概率并按比例进行抽奖
3. **概率处理**：如果所有奖项概率都为0，系统会随机返回一个奖项
4. **排序规则**：奖项排序从1开始，对应转盘上的位置
5. **解锁条件**：转盘2和转盘3通常设置更高的解锁条件
6. **用户次数限制**：设置用户最大次数可以控制转盘使用频率，0表示无限制
7. **规则优先级**：系统会按照规则添加顺序进行匹配
8. **数据备份**：修改配置前建议备份数据库

## 扩展功能

### 可能的扩展
1. **转盘皮肤**：支持不同的转盘外观
2. **活动时间**：设置活动开始和结束时间
3. **用户限制**：限制每个用户的转盘次数
4. **奖品发放**：自动发放奖品到用户账户
5. **数据统计**：转盘使用情况和中奖统计

### 开发建议
1. 使用事务确保数据一致性
2. 添加日志记录便于问题排查
3. 实现缓存机制提高性能
4. 添加数据验证防止异常数据

## 故障排除

### 常见问题
1. **配置不生效**：检查转盘状态是否启用
2. **概率异常**：确认奖项概率总和为1
3. **数据丢失**：检查JSON格式是否正确
4. **页面加载慢**：检查数据库连接和查询性能

### 调试方法
1. 查看浏览器控制台错误信息
2. 检查PHP错误日志
3. 验证数据库连接和表结构
4. 测试API接口返回数据
