## 核心指标说明

### 1. DAU（日活跃用户）
- **计算方式**: 当日注册用户数 + 往日注册但当日活跃的用户数
- **活跃定义**: 当日有游戏交易、充值或提现记录的用户
- **数据来源**: `slot_account`表（注册时间）+ `game_transactions`、`recharge_orders`、`withdraw_orders`表（活跃记录）

### 2. 付费人数
- **计算方式**: 当日付费人数（去重）
- **数据来源**: `recharge_orders`表（已支付订单）
- **筛选条件**: `pay_status = 1`（支付成功）且`created_at`在指定日期范围内

### 3. 付费率
- **计算方式**: 付费人数 / DAU（前端自动转换为百分比）
- **示例**: 假设6月30日活跃用户数：10,000，6月30日付费用户数：500，那么6月30日付费率 = 500/10000 = 0.05（前端显示为5%）

### 4. 付费额
- **计算方式**: 每日付费人数的付费总额
- **数据来源**: `recharge_orders`表（已支付订单的amount字段）
- **保留精度**: 保留2位小数

### 5. ARPU（平均每用户收入）
- **计算方式**: 付费总额 / DAU
- **示例**: 总活跃用户数：10,000，总收入：50,000元，ARPU = 50,000 / 10,000 = 5元
- **保留精度**: 保留2位小数

### 6. ARPPU（平均每付费用户收入）
- **计算方式**: 付费总额 / 付费用户数
- **示例**: 付费用户数：500，总收入：50,000元，ARPPU = 50,000 / 500 = 100元
- **保留精度**: 保留2位小数

### 7. 提现金额
- **计算方式**: 当日用户提现成功总额
- **数据来源**: `withdraw_orders`表（成功提现订单）
- **筛选条件**: `status = 2`（提现成功）且`create_time`在指定日期范围内
- **保留精度**: 保留2位小数

### 8. 提现率
- **计算方式**: 提现用户数 / DAU（前端自动转换为百分比）
- **示例**: 6月30日活跃用户数：10,000，6月30日提现用户数：300，6月30日提现率 = 300/10000 = 0.03（前端显示为3%）

### 9. 下单指标（游戏交易）
- **下单笔数**: 当日游戏下注记录数量（`game_transactions`表中`reason = 'bet'`的记录数）
- **下单人数**: 当日有下注记录的用户数（去重）
- **下单总金额**: 当日下注总金额（取绝对值）
- **现金下单**: 现金钱包下注金额（`wallet_type = 1`）
- **彩金下单**: 彩金钱包下注金额（`wallet_type = 0`）

### 10. 返奖指标
- **返奖金额**: 当日现金赢取总金额（`game_transactions`表中`reason = 'win'`且`wallet_type = 1`的记录）
- **运营商盈利**: 下单金额 - 返奖金额

## 数据模型说明

### 主要数据表
1. **slot_account** - 用户账户表
   - `id`: 用户ID
   - `create_time`: 注册时间（时间戳）
   - `channel_id`: 渠道ID
   - `last_login_time`: 最后登录时间

2. **recharge_orders** - 充值订单表
   - `user_id`: 用户ID
   - `amount`: 充值金额
   - `pay_status`: 支付状态（1=支付成功）
   - `created_at`: 创建时间（时间戳）

3. **withdraw_orders** - 提现订单表
   - `user_id`: 用户ID
   - `amount`: 提现金额
   - `status`: 订单状态（2=提现成功）
   - `create_time`: 创建时间（时间戳）

4. **game_transactions** - 游戏交易表
   - `user_id`: 用户ID
   - `amount`: 交易金额（负数表示下注，正数表示赢取）
   - `real_amount`: 实际金额（赢取时的实际金额）
   - `reason`: 交易原因（'bet'=下注，'win'=赢取）
   - `wallet_type`: 钱包类型（1=现金，0=彩金）
   - `req_time`: 请求时间

5. **slot_operation_data** - 运营数据历史记录表
   - `id`: 主键ID
   - `date`: 统计日期
   - `channel_id`: 渠道ID（null表示全部渠道）
   - `data`: 运营数据JSON
   - `create_time`: 创建时间
   - `update_time`: 更新时间

6. **channel_list** - 渠道列表表
   - `id`: 渠道ID
   - `name`: 渠道名称

## 使用说明

### 访问方式
1. 登录后台管理系统
2. 访问 `/admin/operation` 路径
3. 或通过菜单导航到"运营数据"页面

### 操作步骤
1. **选择日期**: 使用日期选择器选择要查看的日期
2. **选择渠道**: 可选择特定渠道查看数据（可选）
3. **查看数据**: 页面会自动加载并显示对应日期的运营数据
4. **重置筛选**: 点击"重置"按钮可恢复到默认设置

### 数据解读
- **核心指标卡片**: 显示最重要的4个指标，使用不同颜色区分
- **详细指标**: 按功能分组显示所有指标
- **数据表格**: 提供完整的指标列表和说明

## 注意事项

1. **数据准确性**: 确保数据库中的时间字段格式正确
2. **性能优化**: 大量数据查询时可能需要添加索引
3. **时区问题**: 注意服务器时区设置对数据统计的影响
4. **数据一致性**: 确保各表之间的数据关联正确
