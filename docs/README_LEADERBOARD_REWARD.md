# 排行榜奖金池奖励发放功能

## 功能概述

本功能实现了每日、每周、每月排行榜奖金池奖励的自动发放。系统会根据用户排名和配置的奖励比例，从Redis中的奖金池中按比例发放奖励给用户。

## 功能特性

1. **自动发放**：支持每日、每周、每月自动发放奖励
2. **比例分配**：根据用户排名和配置的奖励比例分配奖金池
3. **防重复发放**：通过日志记录防止重复发放
4. **详细日志**：记录发放过程和结果
5. **正确时机**：在正确的时机发放对应周期的奖金池

## 发放时机

- **日榜奖励**：每日零点发放昨日的奖金池
- **周榜奖励**：每周一零点发放上周的奖金池
- **月榜奖励**：每月1号零点发放上月的奖金池

## 数据库表结构

### 排行榜奖励发放日志表 (leaderboard_reward_log)

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int | 主键ID |
| type | enum | 排行榜类型: daily/weekly/monthly |
| pool_amount | decimal(15,2) | 奖金池总金额 |
| distributed_amount | decimal(15,2) | 实际发放金额 |
| success_count | int | 成功发放用户数 |
| fail_count | int | 失败用户数 |
| create_time | bigint | 创建时间 |
| update_time | bigint | 更新时间 |

## 命令使用

### 定时任务

```bash
# 执行定时发放检查（建议每小时执行一次）
php think leaderboard:schedule
```

## 定时任务配置

建议在crontab中配置以下定时任务：

```bash
# 每小时检查一次是否需要发放奖励
0 * * * * cd /path/to/your/project && php think leaderboard:schedule
```

## 奖励发放逻辑

1. **获取奖金池**：从Redis中获取对应类型的奖金池总金额
2. **获取排行榜**：获取当前排行榜数据（按分数降序）
3. **计算奖励**：根据用户排名和配置的奖励比例计算奖励金额
4. **发放奖励**：
   - 使用AccountService的increaseBalance方法更新用户充值钱包余额
   - 记录资金流水日志（使用新的枚举类型）
   - 记录发放统计日志
5. **清理缓存**：清空Redis中的排行榜和奖金池数据

## 资金流水类型

新增了以下资金流水类型：

- `LeaderboardDaily` (29): 排行榜日榜奖励
- `LeaderboardWeekly` (30): 排行榜周榜奖励  
- `LeaderboardMonthly` (31): 排行榜月榜奖励

## 注意事项

1. **数据备份**：发放奖励前建议备份相关数据
2. **监控日志**：关注发放过程中的错误日志
3. **测试环境**：建议先在测试环境验证功能
4. **权限控制**：确保命令执行权限正确
5. **Redis连接**：确保Redis服务正常运行
6. **发放时机**：确保定时任务在正确的时机执行

## 错误处理

系统会记录详细的错误日志，包括：
- 用户不存在
- 余额更新失败
- 资金流水记录失败
- Redis连接异常
- 数据库事务异常
